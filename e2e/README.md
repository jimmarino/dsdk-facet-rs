# E2E Tests

End-to-end tests for Siglet deployed in a Kind cluster with HashiCorp Vault sidecar.

## Prerequisites

- Docker, Kind, kubectl, cargo

## Quick Start

```bash
cd e2e
make all        # setup + test + cleanup
make setup      # one-time cluster setup (Vault, consumer DID, images)
make test-fast  # rebuild images and run tests (dev workflow)
```

## Make Targets

| Target               | Description                                                             |
|----------------------|-------------------------------------------------------------------------|
| `setup`              | Create Kind cluster, deploy Vault, provision consumer DID, build images |
| `build-siglet`       | Full Siglet Docker build                                                |
| `rebuild-siglet`     | Incremental rebuild                                                     |
| `build`              | Build vault-test image                                                  |
| `setup-consumer-did` | (Re)provision consumer DID server — idempotent                          |
| `test`               | Full build + run tests                                                  |
| `test-fast`          | Incremental rebuild + run tests                                         |
| `test-verbose`       | Same as `test` with captured output                                     |
| `cleanup`            | Delete Kind cluster                                                     |

## Consumer DID

The consumer DID keypair is generated by a Rust binary (`consumer-did-setup`) and stored in a Kubernetes Secret (
`consumer-did-private-key`) during `make setup`. The nginx pod serving `http://consumer/.well-known/did.json` is
reconciled on every `make test*` run — no manual intervention needed after initial setup.

To reprovision from scratch, delete the secret and re-run setup:

```bash
kubectl delete secret consumer-did-private-key -n vault-e2e-test
make setup-consumer-did
```

## Test Runner

Supports both `cargo test` and `cargo nextest` (auto-detected). Install nextest for faster parallel runs:

```bash
cargo install cargo-nextest --locked
```

### Run Specific Tests

```bash
# Single test (nextest)
cargo nextest run -p dsdk-facet-e2e-tests --features e2e --run-ignored only -E 'test(test_pull_operations)' --no-capture

# All e2e tests
cargo nextest run -p dsdk-facet-e2e-tests --features e2e --run-ignored only
```
